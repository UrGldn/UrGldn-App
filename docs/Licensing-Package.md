# URGLDN – Licensing qPackage  
*A complete overview of what a bank or credit union receives when licensing URGLDN.*

---

# 1. Overview

URGLDN is a timing-first financial wellness feature designed to integrate directly into a bank or credit union’s existing digital banking experience.  
This Licensing Package outlines exactly what an institution receives when entering a licensing or co‑development agreement.

URGLDN is offered as a **white‑label feature**, allowing institutions to brand and present it as their own.

---

- Vault structure  
- Scheduling logic  
- Release engine behavior

# 3. Licensing Structure

URGLDN offers flexible licensing models depending on the institution’s size and needs.

## 3.1 Standard Licensing Model
- Annual licensing fee  
- Institution receives full white‑label rights  
- Includes documentation, support, and roadmap access  

## 3.2 Royalty‑Based Model
- Lower upfront cost  
- Small per‑active‑user or per‑vault fee  
- Scales with adoption  

## 3.3 Hybrid Model
- Moderate annual fee  
- Reduced per‑user royalty  
- Ideal for mid‑sized institutions  

## 3.4 Pilot‑Only Model
- Short‑term pilot license  
- Limited user base  
- Option to convert to full license  

---

# 4. Institution Responsibilities

To implement URGLDN, the institution must provide:

- API access to internal transfer systems  
- Authentication integration (OAuth)  
- Compliance review  
- UI/UX integration  
- Customer support routing  
- Internal testing resources  

URGLDN does **not** require new accounts or new payment rails — it uses the bank’s existing infrastructure.
